# H# IDLE 增强功能说明

本文档介绍了H# IDLE（集成开发环境）的新增功能和改进。

## 主要功能改进

### 1. 代码编辑功能增强

#### 自动补全
- **关键字补全**：编辑器支持H#语言关键字的自动补全
  - 输入关键字的前几个字母，按下Tab或Enter即可补全
  - 支持的关键字：`let`, `fn`, `return`, `while`, `if`, `else`, `for`, `in`, `print`, `import`, `class`, `extends`, `new`, `private`, `true`, `false`, `and`, `or`, `not`, `interface`, `implements`

#### 括号自动匹配
- **自动补全括号**：输入左括号时自动添加对应的右括号
  - 支持 `()`, `[]`, `{}` 三种括号
  - 光标自动定位到括号中间
- **括号高亮**：当光标位于括号旁边时，自动高亮显示匹配的括号对
  - 匹配的括号对以浅蓝色背景高亮显示
# H# IDLE 快速入门指南

## 界面概览

H# IDLE采用简洁的界面设计，主要包含以下区域：

### 1. 菜单栏（顶部）

简化后的菜单只包含4个核心菜单：

- **File** - 文件管理
  - New Tab (Ctrl+N) - 新建标签页
  - Open Folder - 打开项目文件夹
  - Save (Ctrl+S) - 保存当前文件

- **Edit** - 编辑功能
  - Find (Ctrl+F) - 查找
  - Replace (Ctrl+H) - 替换

- **Run** - 运行控制
  - Compile & Run (F5) - 一键编译并运行
  - Stop (Ctrl+.) - 停止执行

- **Tools** - 工具
  - Shell Mode - REPL交互模式
  - Debug Mode - 调试模式

### 2. 工具栏（编辑器上方）

精简为5个常用按钮：

- 📁 **Open Folder** - 打开项目文件夹
- 💾 **Save** - 保存文件
- ▶ **Compile & Run** - 一键编译并运行（绿色高亮，最常用）
- ⏹ **Stop** - 停止运行
- 🗑 **Clear** - 清空输出

### 3. 文件浏览器（左侧）

显示项目文件树结构：
- 双击文件即可在新标签页中打开
- 点击刷新按钮更新文件列表
- 自动显示.hto、.hbc、.py源文件

### 4. 代码编辑器（中央）

支持多标签页的代码编辑器：
- 语法高亮
- 自动补全
- 括号匹配
- 行号显示
- 断点设置（点击行号区域）

### 5. 输出面板（底部）

显示程序运行结果和消息。

## 快速开始

### 方式1：打开文件夹（推荐）

1. 点击 **📁 Open Folder** 按钮
2. 选择你的项目文件夹
3. 在左侧文件树中双击要编辑的文件
4. 按 **F5** 或点击 **▶ Compile & Run** 运行

### 方式2：单个文件

1. File -> New Tab 创建新文件
2. 编写代码
3. 按 **F5** 运行（会提示保存）
4. 或直接点击 **▶ Compile & Run**

2. 在IDLE中打开该文件夹

## 提示

- **Compile & Run** 按钮是绿色的，表示这是最常用的功能
- 点击行号左侧可以设置/取消断点（红色圆点）
- 启用Debug Mode可以看到变量和断点面板
- Shell Mode提供REPL交互式编程环境

## 与旧版对比

**改进点：**
- ✓ 菜单从5个减少到4个，更清晰
- ✓ 工具栏从11个按钮减少到5个，更简洁
- ✓ 突出显示最常用的"Compile & Run"功能
- ✓ 保留了所有核心功能，只是隐藏了不常用的
- ✓ 界面更现代化，操作更直观

**移除的功能（可通过其他方式访问）：**
- 单独的"Open File" - 使用文件树双击打开
- "Run Source" - 统一使用"Compile & Run"
- "Run Selection" - 较少使用
- "Compile .hbc" - 包含在Compile & Run中
- "Run .hbc" - 较少使用
- "Syntax Check" - Compile & Run会自动检查

#### 自动缩进
- **智能缩进**：按Enter键时自动保持上一行的缩进级别
- **块级缩进**：在 `{` 后按Enter会自动增加一级缩进（4个空格）

#### 查找和替换
- **查找对话框**：Edit -> Find (快捷键: Ctrl+F / Cmd+F)
  - 支持大小写敏感/不敏感搜索
  - 自动循环搜索（到达文件末尾后从头开始）
- **替换对话框**：Edit -> Replace (快捷键: Ctrl+H / Cmd+H)
  - 支持查找并替换文本
  - 同样支持大小写选项

### 2. 项目管理功能

#### 打开文件夹
- **文件夹浏览**：File -> Open Folder 或点击 "Open Folder" 按钮
  - 选择项目文件夹后自动加载文件树
  - 窗口标题显示当前文件夹路径
  - 快速切换不同项目

#### 多标签页支持
  - 同样支持大小写选项

### 2. 调试功能

#### 断点管理
- **设置断点**：点击行号区域可在该行设置/取消断点
  - 断点以红色圆点显示
  - 断点信息会显示在调试面板中
- **断点显示**：断点在行号左侧以醒目的红点标识

#### 调试面板
- **变量查看**：显示当前作用域内的变量名、值和类型
- **断点列表**：显示所有已设置的断点及其位置
- **调试控制按钮**：
  - Step Into（单步进入）
  - Step Over（单步跳过）
  - Continue（继续执行）

#### 调试模式切换
- 通过 Tools -> Debug Mode 启用/禁用调试模式
- 启用调试模式后自动显示调试面板和控制按钮

### 3. 项目管理功能

#### 多标签页支持
- **多文件编辑**：支持同时打开多个文件，每个文件在一个标签页中
- **新建标签页**：File -> New Tab (快捷键: Ctrl+N / Cmd+N)
- **关闭标签页**：点击标签页上的关闭按钮（至少保留一个标签页）
- **标签页切换**：点击标签页或使用Ctrl+Tab切换

#### 文件浏览器
- **项目文件树**：左侧面板显示当前项目的文件结构
  - 递归显示子目录（最多2层深度）
  - 只显示相关源文件（.hto, .hbc, .py）
  - 自动过滤隐藏文件和系统目录
- **快速打开**：双击文件树中的文件即可在新标签页中打开
- **刷新功能**：点击文件浏览器顶部的刷新按钮更新文件列表

#### 文件操作改进
- **智能打开**：打开文件时检查是否已在其他标签页中打开，避免重复
- **自动标签命名**：标签页名称自动使用文件名
- **保存跟踪**：保存文件时自动更新标签页名称和文件路径

### 4. 界面改进

#### 布局优化
- **可调整分隔条**：文件浏览器和编辑器之间使用可拖动的分隔条
  - 初始宽度：文件浏览器250px，编辑器区域自适应
  - 可拖动调整两个区域的宽度比例
- **更大的默认窗口**：窗口尺寸从900x600增加到1200x700

#### 菜单增强
- **新增Edit菜单**：包含Find和Replace功能
- **File菜单增强**：添加New Tab和Open Folder选项

### 5. 一键编译运行

#### Compile & Run功能
- **一键操作**：点击 "Compile & Run" 按钮或使用 Run -> Compile & Run (快捷键: F5)
  - 自动将源代码编译为字节码
  - 立即执行编译后的字节码
  - 在输出面板显示编译和运行状态
  - 比直接运行源码更高效（跳过解释器）

#### 工作流程
1. 编写H#代码
2. 点击 "Compile & Run" 或按F5
3. 系统自动：
   - 词法分析
   - 语法解析
   - 编译为字节码
   - 执行字节码
4. 查看输出结果

**优势**：
- 更快的执行速度（使用编译后的字节码）
- 自动检测编译错误
- 清晰的编译和运行状态提示

### 6. 代码高亮器改进

现有的语法高亮器保持不变，支持：
- 关键字高亮（蓝色粗体）
- 数字高亮（深红色）
- 字符串高亮（绿色）
- 注释高亮（灰色）
- 函数名高亮（紫色斜体）
- 类名高亮（青色粗体）
- 多行注释 `/* ... */`
- 多行字符串 `"""..."""` 和 `'''...'''`

## 使用示例

### 基本工作流程

1. **启动IDLE**：
   ```bash
   python3 idle_pyqt.py
   ```

2. **打开项目文件**：
   - 使用 File -> Open 打开单个文件
   - 或在文件浏览器中双击文件

3. **编辑代码**：
   - 利用自动补全快速输入关键字
   - 括号自动匹配减少输入错误
   - 自动缩进保持代码整洁

4. **查找和替换**：
   - 使用 Edit -> Find 搜索文本
   - 使用 Edit -> Replace 批量替换

5. **调试程序**：
   - 启用 Tools -> Debug Mode
   - 在行号区域点击设置断点
   - 使用调试控制面板执行程序

6. **运行代码**：
   - 点击 "Run Source" 通过解释器运行
   - 点击 "Compile & Run" 或按F5一键编译并运行（推荐）
   - 输出显示在底部Output区域

- **Ctrl+F / Cmd+F**：查找
- **Ctrl+H / Cmd+H**：替换
- **F5**：一键编译并运行（Compile & Run）

## 技术实现细节

### CodeEditor类增强
- 继承自QPlainTextEdit
- 添加了Completer支持
- 重写了keyPressEvent处理特殊按键
- 实现了括号匹配和高亮
- 支持断点存储和显示

### IDLEWindow类重构
- 使用QTabWidget管理多个编辑器实例
- 使用QSplitter实现可调整的面板布局
- 使用QTreeWidget实现文件浏览器
- 添加了调试面板和支持组件

### 兼容性
- 完全兼容现有的H#解释器和编译器
- 保持了所有原有功能
- 向后兼容现有的.hto文件

## 故障排除

### 常见问题

1. **IDLE无法启动**：
   - 确保PyQt5已正确安装：`pip install PyQt5`
   - 检查Python版本（需要Python 3.6+）

2. **自动补全不工作**：
   - 确保输入的是完整的关键字前缀
   - 尝试重新聚焦编辑器窗口

3. **文件浏览器为空**：
   - 点击刷新按钮更新文件列表
   - 确保当前目录下有.hto、.hbc或.py文件

4. **断点不显示**：
   - 确保点击的是行号区域（左侧灰色区域）
   - 启用了Debug Mode才能在调试面板中看到断点列表
